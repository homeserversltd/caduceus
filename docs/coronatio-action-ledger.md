# Coronatio GUI action ledger

Scope: observed Coronatio GUI controls at `origin/main` during GUI–CLI parity Slice 0. This is an action inventory, not an assertion that a visible control has an admitted actuator. “Inert” means the Crown deliberately does not dispatch; “un-wired” means it has a visible route but no declared Caduceus registry entry.

Registry rule: only `restart-website` is registered in Slice 0, and it is **intentionally unavailable**. Its canonical CLI (`caduceus service restart coronatio`), canonical HTTP route (`POST /api/v1/service/coronatio/restart` with `{}`), and temporary legacy alias resolve the same definition. They return an explicit non-success receipt with `mutationPerformed:false`; Slice 1 owns actuator/readback wiring.
